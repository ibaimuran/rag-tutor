## 第2章 提示词优化技巧

### 2.1 链式思考与逐步推理

#### 1. 核心概念

想象一下，你正在教一个小朋友解一道复杂的数学应用题。你不会直接把答案告诉他，而是会引导他：“我们先读一遍题目，看看它问的是什么？然后，我们找出题目中给了哪些数字？接着，想一想应该用加法还是减法？最后，一步步算出结果。”

**链式思考与逐步推理**，正是将这种“分步思考、逐步解决”的思维方式，教给人工智能模型。它的核心是：**不要直接要求AI给出最终答案，而是引导它像人类一样，把解决问题的逻辑过程、中间步骤和推理链条，一步一步地“说出来”或“写出来”**。

这就像给AI搭建了一个思考的“脚手架”，让它从“A”到“B”再到“C”，而不是直接从“A”跳到“C”。通过显式地展示推理过程，AI不仅能得出更准确的答案，其思考路径也变得透明、可解释，我们还能从中发现它哪里想错了。

#### 2. 关键定义

- **链式思考 (Chain-of-Thought, CoT)**：一种提示词技术，通过在提示中要求模型**生成一系列中间推理步骤**，来引导模型解决复杂问题。它模拟了人类解决问题的逻辑链条。
- **逐步推理 (Step-by-Step Reasoning)**：与链式思考紧密相关，强调将复杂的任务**分解为多个简单、连续的步骤**，并逐一解决。这是实现链式思考的具体方法。
- **中间步骤 (Intermediate Steps)**：在最终答案之前，模型生成的、用于连接问题和答案的**逻辑推理过程**。例如，在计算“小明有5个苹果，吃了2个，又买了3个，现在有几个？”时，“5 - 2 = 3”和“3 + 3 = 6”就是中间步骤。
- **思维链提示 (Chain-of-Thought Prompting)**：在提示词中**提供一个或多个包含完整推理过程的示例**，让模型学习并模仿这种“先推理，后回答”的模式。这是实现链式思考最有效的方式之一。

#### 3. 示例与应用

**示例一：解决数学应用题**

- **普通提示**：`问：一个农场有12只鸡，鸭的数量是鸡的3倍，鹅比鸭少5只。农场一共有多少只家禽？`
- **链式思考提示**：`问：一个农场有12只鸡，鸭的数量是鸡的3倍，鹅比鸭少5只。农场一共有多少只家禽？让我们一步一步思考。`
- **AI输出（链式思考）**：
    1.  首先，计算鸭的数量：鸭是鸡的3倍，所以鸭有 `12 * 3 = 36` 只。
    2.  然后，计算鹅的数量：鹅比鸭少5只，所以鹅有 `36 - 5 = 31` 只。
    3.  最后，计算总数：鸡 + 鸭 + 鹅 = `12 + 36 + 31 = 79` 只。
    所以，农场一共有79只家禽。

**示例二：逻辑推理题**

- **普通提示**：`所有A都是B，所有B都是C。那么，所有A都是C吗？`
- **链式思考提示**：`所有A都是B，所有B都是C。那么，所有A都是C吗？请逐步推理。`
- **AI输出（链式思考）**：
    1.  前提1：所有A都是B。这意味着如果某物是A，它一定是B。
    2.  前提2：所有B都是C。这意味着如果某物是B，它一定是C。
    3.  结合前提：如果某物是A，根据前提1，它一定是B。既然它是B，根据前提2，它一定是C。
    4.  结论：因此，所有A都是C。这个推理是有效的。

**示例三：生活决策**

- **场景**：你想决定是否要买一辆昂贵的电动车。
- **链式思考提示**：`帮我分析一下是否应该买一辆价值30万的电动车。请列出利弊，并逐步推理。`
- **AI输出**：
    1.  **优点分析**：首先，长期来看，电费比油费便宜很多，假设每年开2万公里，能省下约1万元油费。其次，电动车保养简单，每年能省下2000元。第三，环保，符合未来趋势。
    2.  **缺点分析**：第一，初始购车成本高，比同级别燃油车贵5-10万。第二，充电不如加油方便，长途旅行有里程焦虑。第三，电池有衰减风险，几年后可能需要更换，成本不菲。
    3.  **综合推理**：如果我的年收入较高（>50万），且家里有固定充电桩，主要用于城市通勤，那么优点（省钱、环保）会逐渐抵消缺点，可以考虑购买。如果收入一般，且经常跑长途，那么缺点（成本高、充电不便）会更突出，不建议购买。

#### 4. 常见误区

- **误区一：认为“链式思考”就是让AI“自言自语”或“说废话”。**
    - **纠正**：链式思考不是无意义的填充，而是**有逻辑、有目标的推理过程**。每一步推理都应该服务于最终答案，是解决问题链条上不可或缺的一环。如果AI只是重复问题或说“然后我们思考一下”，而没有实质性的逻辑推导，那就不是有效的链式思考。

- **误区二：认为只要在提示词末尾加上“让我们一步一步思考”就万事大吉。**
    - **纠正**：虽然这句话很有效，但它的威力取决于问题的复杂度。对于简单问题，它可能足够。但对于非常复杂的问题，**提供1-2个完整的“思维链”示例（Few-shot CoT）** 效果会更好。模型需要看到“好”的推理过程是什么样的，才能模仿学习。仅仅一句指令，有时不足以引导模型进行正确的逻辑分解。
