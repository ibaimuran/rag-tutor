## 第2章 流程控制

### 2.4 循环嵌套与综合应用

#### 1. 核心概念

想象一下，你需要打扫一栋有3层楼的房子，每层楼有4个房间。你的任务是：**先打扫完第1层的所有房间，再去打扫第2层，最后打扫第3层**。

这个任务就体现了**循环嵌套**的思想。我们可以把“打扫一层楼”看作一个**外层循环**（总共循环3次），而“打扫这一层楼里的每个房间”看作一个**内层循环**（每层循环4次）。

在编程中，**循环嵌套**指的是在一个循环结构（如 `for` 或 `while`）的**内部**，完整地包含另一个循环结构。外层循环每执行一次，内层循环就会从头到尾完整地执行一遍。这种结构非常适合处理具有“行”和“列”关系的数据，或者需要分步骤、分层次完成的任务。

#### 2. 关键定义

-   **循环嵌套 (Nested Loop)**：指一个循环体内部包含另一个完整的循环结构。
-   **外层循环 (Outer Loop)**：位于外部的循环，控制整体的执行轮次或大的步骤。
-   **内层循环 (Inner Loop)**：位于外部的循环体内部，负责完成每个大步骤中的小细节。
-   **循环体 (Loop Body)**：循环结构中，被重复执行的代码块。在嵌套循环中，内层循环整体就是外层循环的循环体的一部分。

#### 3. 示例与应用

**示例1：打印九九乘法表**

这是一个经典的嵌套循环应用。外层循环控制“行数”（从1到9），内层循环控制“列数”（从1到当前行数）。

```python
# 外层循环：控制行
for i in range(1, 10):
    # 内层循环：控制列，列数不超过当前行数
    for j in range(1, i + 1):
        # 打印乘法算式，并用 \t 制表符对齐
        print(f"{j} * {i} = {i * j}", end="\t")
    # 每打印完一行后换行
    print()
```

**示例2：模拟日程安排**

假设你有一周（7天）的日程，每天需要完成3项任务。外层循环遍历每一天，内层循环遍历当天的每一项任务。

```python
days = ["周一", "周二", "周三", "周四", "周五", "周六", "周日"]
tasks = ["晨跑", "学习编程", "阅读"]

# 外层循环：遍历每一天
for day in days:
    print(f"--- {day} 的日程 ---")
    # 内层循环：遍历当天的任务
    for task in tasks:
        print(f"  执行任务：{task}")
    print()  # 每天结束后空一行
```

**示例3：查找列表中的重复元素**

假设有一个学生名单，我们需要找出所有重名的学生。外层循环遍历每个学生，内层循环从当前学生的下一个位置开始，查找是否有相同的名字。

```python
students = ["小明", "小红", "小刚", "小明", "小丽", "小红"]
found_duplicates = []

# 外层循环：遍历每个学生
for i in range(len(students)):
    # 内层循环：从 i+1 开始，查找与 students[i] 相同的名字
    for j in range(i + 1, len(students)):
        if students[i] == students[j]:
            # 避免重复添加
            if students[i] not in found_duplicates:
                found_duplicates.append(students[i])

print("重复的名字有：", found_duplicates)
```

#### 4. 常见误区

-   **误区一：混淆内外层循环的控制变量**
    初学者容易在编写内层循环时，错误地使用了外层循环的循环变量，导致逻辑错误。例如，在打印九九乘法表时，内层循环的 `range(1, i + 1)` 必须使用外层变量 `i` 来控制列数，如果错误地写成了 `range(1, j + 1)` 或固定值，结果就会出错。

-   **误区二：认为内层循环只执行一次**
    这是最根本的错误。请牢记：**外层循环每执行一次，内层循环都会完整地从头到尾执行一遍**。例如，外层循环执行3次，内层循环执行4次，那么内层循环的代码总共会被执行 `3 * 4 = 12` 次。
