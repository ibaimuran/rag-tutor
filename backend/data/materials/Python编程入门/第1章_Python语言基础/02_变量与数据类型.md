## 第1章 Python语言基础

### 1.1 变量与数据类型

#### 1. 核心概念

想象一下，你有一个神奇的储物箱。你可以往里面放任何东西：一本书、一个苹果、一张写有数字的纸条，甚至是一段音乐。这个储物箱就是**变量**，而里面存放的东西就是**数据**。

在Python中，**变量**就是用来存放数据的“容器”。我们可以给这个容器起一个名字，比如 `my_book`，然后通过这个名字来使用或修改里面的数据。

但是，不同的数据有不同的“性质”。比如，数字可以加减乘除，文字可以拼接，而音乐不能直接和数字相加。Python为了区分这些不同的“性质”，引入了**数据类型**的概念。数据类型告诉Python：“这个变量里装的是什么类型的东西，应该怎么处理它。”

简单来说，**变量是存放数据的盒子，数据类型是盒子上贴的标签，告诉我们里面是什么东西**。

#### 2. 关键定义

-   **变量 (Variable)**：在计算机内存中用于存储数据的**命名**位置。你可以通过变量名来访问和操作存储在其中的数据。
-   **赋值 (Assignment)**：使用等号 `=` 将数据放入变量的过程。例如 `age = 25`，意思是将整数 `25` 这个数据，放入名为 `age` 的变量中。
-   **数据类型 (Data Type)**：定义数据在计算机中的表示方式和可执行的操作。Python中常见的基本数据类型包括：
    -   **整数 (int)**：没有小数点的数字，如 `10`, `-5`, `0`。
    -   **浮点数 (float)**：有小数点的数字，如 `3.14`, `-0.5`, `2.0`。
    -   **字符串 (str)**：用单引号 `'...'` 或双引号 `"..."` 括起来的文本，如 `"Hello"`, `'Python'`。
    -   **布尔值 (bool)**：表示逻辑状态，只有两个值：`True` (真) 和 `False` (假)。
-   **动态类型 (Dynamic Typing)**：Python的一个特性，意味着你不需要提前声明变量的数据类型。Python会在你赋值时自动判断并“记住”数据的类型。同一个变量可以先后存放不同类型的数据。

#### 3. 示例与应用

**生活例子1：记录个人信息**

你正在填写一个在线表格，需要记录你的名字、年龄和是否喜欢编程。

```python
# 创建一个变量来存储你的名字（字符串）
name = "小明"

# 创建一个变量来存储你的年龄（整数）
age = 18

# 创建一个变量来存储你是否喜欢编程（布尔值）
likes_programming = True

# 打印这些信息
print("姓名：", name)
print("年龄：", age)
print("喜欢编程：", likes_programming)
```

**生活例子2：计算购物总价**

你去超市买了苹果和香蕉，需要计算总价。

```python
# 苹果单价（浮点数）和数量（整数）
apple_price = 5.5
apple_quantity = 3

# 香蕉单价（浮点数）和数量（整数）
banana_price = 4.0
banana_quantity = 2

# 计算总价（浮点数）
total_cost = apple_price * apple_quantity + banana_price * banana_quantity

# 打印总价
print("总价为：", total_cost, "元")
```

**生活例子3：拼接问候语**

你想用程序生成一句个性化的问候语。

```python
# 存储用户的名字（字符串）
user_name = "小红"

# 拼接字符串，生成问候语
greeting = "你好，" + user_name + "！欢迎学习Python。"

# 打印问候语
print(greeting)
```

#### 4. 常见误区

-   **误区一：变量名可以以数字开头。**
    -   **正确理解**：变量名只能以**字母**（a-z, A-Z）或**下划线** `_` 开头。例如，`1st_place` 是错误的，而 `first_place` 或 `_1st_place` 是正确的。

-   **误区二：`=` 是“等于”的意思。**
    -   **正确理解**：在编程中，单个等号 `=` 是**赋值运算符**，表示“把右边的值赋给左边的变量”。例如 `x = x + 1` 在数学上不成立，但在编程中意思是“取出 `x` 当前的值，加1，再把结果存回 `x`”。判断是否相等需要使用双等号 `==`。

#### 5. 思考题

Python是动态类型语言，这意味着变量可以随时改变其数据类型。例如：

```python
my_var = 10      # 此时 my_var 是整数类型
my_var = "Hello" # 此时 my_var 变成了字符串类型
```

这种灵活性带来了便利，但也可能引发一些意想不到的错误。请思考：在什么情况下，这种“动态类型”的特性可能会导致程序出现难以发现的错误？你可以结合生活中的例子来思考。
