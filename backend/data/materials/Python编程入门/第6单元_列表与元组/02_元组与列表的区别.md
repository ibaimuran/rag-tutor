## 第6单元 列表与元组

### 1. 核心概念

想象一下，你有一个**购物清单**和一个**身份证信息**。

*   **购物清单**（就像**列表**）：你可以随时往上面添加新商品（比如“牛奶”），也可以划掉已经买到的商品（比如“鸡蛋”），甚至可以把“面包”换成“蛋糕”。它是一个**可以变化**的清单。
*   **身份证信息**（就像**元组**）：你的姓名、性别、出生日期一旦被印在身份证上，就**不能随意更改**了。你不能把“张三”改成“李四”，也不能把出生年份改掉。它是一个**固定不变**的记录。

在Python中，**列表（List）** 和**元组（Tuple）** 都是用来存储一系列数据的有序容器。它们最核心的区别就在于：**列表是可变的（Mutable），而元组是不可变的（Immutable）**。这意味着，列表创建后可以增、删、改其中的元素，而元组一旦创建，其内容就“锁定”了，无法进行任何修改。

### 2. 关键定义

*   **列表（List）**：一种**有序**、**可变**的数据集合。使用方括号 `[]` 定义，元素之间用逗号分隔。例如：`my_list = [1, 2, 3]`。
*   **元组（Tuple）**：一种**有序**、**不可变**的数据集合。使用圆括号 `()` 定义，元素之间用逗号分隔。例如：`my_tuple = (1, 2, 3)`。
*   **可变性（Mutability）**：指对象创建后，其内容（元素的值、数量、顺序）是否可以被改变。列表支持，元组不支持。
*   **不可变性（Immutability）**：指对象创建后，其内容**无法被改变**。任何试图修改元组的操作都会导致错误。

### 3. 示例与应用

1.  **学生成绩管理系统（列表）**
    *   场景：老师需要记录一个班级的考试成绩，并随时更新。
    *   代码示例：
        ```python
        # 使用列表存储成绩，因为成绩会变动
        scores = [85, 92, 78, 95]
        print("原始成绩:", scores)

        # 修改成绩：将78改为88
        scores[2] = 88
        print("修改后成绩:", scores)

        # 添加新成绩
        scores.append(100)
        print("添加后成绩:", scores)
        ```
    *   解释：成绩是动态变化的，需要增删改，所以用**列表**。

2.  **地球的经纬度坐标（元组）**
    *   场景：一个地点的经纬度是固定不变的，比如北京天安门广场的坐标。
    *   代码示例：
        ```python
        # 使用元组存储坐标，因为坐标是固定的
        beijing_coords = (39.9042, 116.4074)
        print("北京坐标:", beijing_coords)

        # 尝试修改坐标（会报错！）
        # beijing_coords[0] = 40.0  # 这行代码会引发 TypeError
        ```
    *   解释：坐标是地理上的固定值，不应该被程序意外修改，用**元组**更安全。

3.  **函数的多个返回值（元组）**
    *   场景：一个函数需要同时返回“最大值”和“最小值”。
    *   代码示例：
        ```python
        def find_min_max(numbers):
            return min(numbers), max(numbers)  # 返回一个元组

        result = find_min_max([3, 7, 1, 9])
        print("结果:", result)  # 输出: (1, 9)
        print("最小值:", result[0])  # 输出: 1
        print("最大值:", result[1])  # 输出: 9
        ```
    *   解释：函数返回一组相关的、不希望被修改的数据时，使用**元组**非常方便。

### 4. 常见误区

*   **误区一：认为元组完全不能“变”**
    *   错误理解：元组里的任何东西都不能变。
    *   正确理解：元组的**元素引用**不可变。如果元组里包含一个**列表**，这个列表本身的内容是可以修改的。
    *   示例：
        ```python
        my_tuple = (1, [2, 3], 4)
        # my_tuple[0] = 10  # 错误！不能修改元组元素
        my_tuple[1][0] = 99  # 正确！修改了元组内列表的元素
        print(my_tuple)  # 输出: (1, [99, 3], 4)
        ```
        这里，元组中第二个元素的“引用”始终指向那个列表对象，但我们修改了列表对象内部的数据，这是允许的。

*   **误区二：混淆括号**
    *   错误理解：看到圆括号就认为是元组。
    *   正确理解：圆括号 `()` 也可以用于表达式分组或函数调用。定义只有一个元素的元组时，必须加一个逗号。
    *   示例：
        ```python
        a = (5)    # 这是整数5，不是元组
        b = (5,)   # 这才是只有一个元素5的元组
        print(type(a))  # 输出: <class 'int'>
        print(type(b))  # 输出: <class 'tuple'>
        ```
