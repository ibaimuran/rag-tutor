## 第3章 数据结构

### 3.2 元组与字典

#### 1. 核心概念

想象一下，你有一个“只读”的购物清单，上面写好了要买的东西，一旦写好就不能再修改。在Python中，**元组（Tuple）** 就是这样一个“只读”的序列，它用来存储一组不可变的数据。

再想象一下，你有一本“电话簿”，你可以通过“姓名”这个关键词，快速找到对应的“电话号码”。在Python中，**字典（Dictionary）** 就是这样一个“键值对”的集合，它允许你通过一个唯一的“键（Key）”来快速访问与之关联的“值（Value）”。

简单来说，元组是“不能改动的列表”，而字典是“有索引的映射表”。

#### 2. 关键定义

- **元组（Tuple）**：一个**有序**、**不可变**的序列。它使用圆括号 `()` 定义，元素之间用逗号 `,` 分隔。一旦创建，其内容不能被添加、删除或修改。
- **字典（Dictionary）**：一个**无序**、**可变**的映射类型。它使用花括号 `{}` 定义，内部由**键（Key）** 和**值（Value）** 组成的“键值对”构成，键与值之间用冒号 `:` 分隔，键值对之间用逗号 `,` 分隔。
- **键（Key）**：字典中用于索引的**唯一**标识符，必须是**不可变**类型（如字符串、数字、元组）。
- **值（Value）**：与键相关联的数据，可以是任意数据类型（包括列表、字典等可变类型）。

#### 3. 示例与应用

**生活例子1：坐标点（元组）**

在地图上，一个地点的经纬度坐标是固定不变的。例如，北京天安门的坐标可以表示为 `(39.9042, 116.4074)`。这个坐标一旦确定，就不应该被修改，非常适合用**元组**来存储。

```python
# 定义一个元组表示天安门坐标
tiananmen_coordinate = (39.9042, 116.4074)
print(tiananmen_coordinate)  # 输出: (39.9042, 116.4074)

# 尝试修改元组会报错
# tiananmen_coordinate[0] = 40.0  # 这行代码会引发 TypeError
```

**生活例子2：学生信息表（字典）**

学校需要记录学生的信息，比如学号、姓名、成绩。我们可以用“学号”作为**键**，用包含姓名和成绩的**字典**作为**值**，快速查找。

```python
# 定义一个字典，键是学号，值是另一个字典（包含姓名和成绩）
student_info = {
    "001": {"name": "张三", "score": 95},
    "002": {"name": "李四", "score": 88}
}

# 通过学号快速查找学生信息
print(student_info["001"])  # 输出: {'name': '张三', 'score': 95}
print(student_info["002"]["name"])  # 输出: 李四
```

**应用场景3：配置参数（字典）**

在软件开发中，经常需要存储程序的配置信息，比如数据库地址、用户名、密码等。这些信息通常以“键值对”的形式存在，非常适合用**字典**来管理。

```python
# 定义一个字典存储数据库配置
db_config = {
    "host": "localhost",
    "port": 3306,
    "user": "admin",
    "password": "123456"
}

# 获取配置信息
print(f"数据库地址: {db_config['host']}:{db_config['port']}")
```

#### 4. 常见误区

- **误区一：混淆元组与列表的括号**
  - **错误理解**：认为元组用圆括号 `()`，列表用方括号 `[]`，但功能完全一样。
  - **正确理解**：元组是**不可变**的，列表是**可变**的。这是两者最本质的区别。例如，`my_tuple = (1, 2, 3)` 不能修改元素，而 `my_list = [1, 2, 3]` 可以。

- **误区二：字典的键可以重复或使用可变类型**
  - **错误理解**：认为字典的键可以重复，或者可以使用列表作为键。
  - **正确理解**：字典的**键必须是唯一的**，如果重复，后面的值会覆盖前面的。同时，**键必须是不可变类型**（如字符串、数字、元组），因为可变类型（如列表）无法被哈希，不能作为键。
