## 第3章 数据结构

### 3.1 列表 (List)

#### 1. 核心概念

想象一下，你有一个购物清单，上面按顺序写着“牛奶”、“面包”、“鸡蛋”。这个清单就是一个**列表**。在Python中，**列表 (List)** 就是用来按顺序存放多个数据项的“容器”。它非常灵活，可以装下任何类型的东西，比如数字、文字，甚至其他列表。列表里的每个项目都有一个固定的位置编号，我们称之为**索引 (Index)**，通过这个编号我们可以快速找到并操作对应的项目。

#### 2. 关键定义

-   **列表 (List)**：一种有序的、可变的、可以包含任意类型元素的**数据结构**。在Python中，用方括号 `[]` 表示，元素之间用逗号 `,` 分隔。例如：`my_list = [1, "hello", 3.14]`。
-   **元素 (Element)**：列表中存储的每一个独立的数据项。例如，在 `[10, 20, 30]` 中，`10`、`20`、`30` 都是元素。
-   **索引 (Index)**：列表中每个元素的位置编号。**索引从0开始**，即第一个元素的索引是0，第二个是1，以此类推。例如，`my_list[0]` 访问第一个元素。Python还支持**负索引**，`-1` 表示最后一个元素，`-2` 表示倒数第二个。
-   **可变性 (Mutable)**：列表创建后，可以修改、添加或删除其中的元素，这是列表区别于**元组 (Tuple)** 的一个重要特性。

#### 3. 示例与应用

1.  **生活例子：学生成绩单**
    -   场景：老师需要记录一个班级里5名学生的数学成绩。
    -   Python实现：`scores = [95, 87, 92, 78, 88]`
    -   操作：我们可以通过索引访问某个学生的成绩，比如 `scores[2]` 得到第三位同学的成绩92。也可以添加新同学的成绩：`scores.append(100)`。

2.  **生活例子：待办事项清单**
    -   场景：你有一个每日任务清单，需要按优先级排序。
    -   Python实现：`todo_list = ["写报告", "开会", "回复邮件", "健身"]`
    -   操作：完成“写报告”后，可以删除它：`todo_list.remove("写报告")`。或者，你发现“健身”最重要，可以把它插入到最前面：`todo_list.insert(0, "健身")`。

3.  **应用场景：游戏中的角色背包**
    -   场景：一个角色游戏，玩家有一个背包，可以存放各种道具。
    -   Python实现：`inventory = ["剑", "盾牌", "药水", "地图"]`
    -   操作：玩家捡到一把“弓”，可以用 `inventory.append("弓")` 放入背包。当使用“药水”后，可以用 `inventory.pop(2)` 移除它。

#### 4. 常见误区

-   **误区一：混淆索引从0开始**
    -   **错误理解**：认为列表 `my_list = [10, 20, 30]` 的第一个元素是 `my_list[1]`。
    -   **正确理解**：索引从0开始，所以第一个元素是 `my_list[0]`，值为10。最后一个元素是 `my_list[2]`，值为30。

-   **误区二：试图访问不存在的索引**
    -   **错误理解**：对于只有3个元素的列表 `my_list = [10, 20, 30]`，尝试访问 `my_list[3]`。
    -   **正确理解**：列表的索引范围是0到 `len(my_list)-1`。访问超出范围的索引会引发 `IndexError` 错误。应该先检查列表长度，或者使用循环安全地访问。
