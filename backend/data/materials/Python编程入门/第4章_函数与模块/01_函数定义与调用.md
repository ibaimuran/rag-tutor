## 第4章 函数与模块

### 4.1 函数定义与调用

#### 1. 核心概念

想象一下，你正在准备一顿丰盛的晚餐。你需要洗菜、切菜、炒菜、煮饭……如果每做一道菜，你都要重新洗一遍锅、重新切一遍葱姜蒜，那会非常麻烦和低效。

在编程中，**函数** 就像是一个预先准备好的“烹饪步骤”或“小工具”。它把一段完成特定功能的代码（比如“计算两个数的和”或“打印欢迎信息”）打包在一起，并给它取一个名字。当你需要执行这个功能时，只需要“调用”这个函数的名字，程序就会自动执行打包好的代码，而不需要你每次都重新写一遍。

简单来说，**函数定义** 就是“创造”这个工具的过程，而**函数调用** 就是“使用”这个工具的过程。

#### 2. 关键定义

-   **函数 (Function)**：一段具有特定功能、可重复使用的代码块。它接受输入（**参数**），进行处理，并可能返回输出（**返回值**）。
-   **函数定义 (Function Definition)**：使用 `def` 关键字创建一个新函数的过程。它指定了函数名、参数列表以及要执行的代码块。
-   **函数调用 (Function Call)**：通过函数名和括号 `()` 来执行函数中代码的过程。
-   **参数 (Parameter)**：在函数定义时，写在括号里的变量，用于接收调用时传入的数据。
-   **实参 (Argument)**：在函数调用时，实际传递给函数的值。
-   **返回值 (Return Value)**：函数执行完毕后，使用 `return` 语句返回给调用者的结果。如果没有 `return`，函数默认返回 `None`。

#### 3. 示例与应用

**示例一：制作咖啡的步骤**

假设我们想模拟制作一杯咖啡的过程。

```python
# 1. 函数定义：定义“制作咖啡”这个步骤
def make_coffee(sugar, milk):
    """
    这个函数模拟制作一杯咖啡。
    参数:
        sugar (int): 糖的份数
        milk (bool): 是否需要加奶
    """
    print("开始制作咖啡...")
    print(f"加入 {sugar} 份糖")
    if milk:
        print("加入牛奶")
    print("一杯香浓的咖啡做好了！")
    # 没有 return 语句，函数执行完毕返回 None

# 2. 函数调用：使用这个步骤
make_coffee(2, True)  # 制作一杯加2份糖和牛奶的咖啡
print("---")
make_coffee(0, False) # 制作一杯不加糖不加奶的黑咖啡
```

**输出结果：**
```
开始制作咖啡...
加入 2 份糖
加入牛奶
一杯香浓的咖啡做好了！
---
开始制作咖啡...
加入 0 份糖
一杯香浓的咖啡做好了！
```

**示例二：计算圆的面积**

这是一个带有返回值的函数。

```python
import math

# 1. 函数定义：计算圆的面积
def calculate_circle_area(radius):
    """
    计算并返回圆的面积。
    参数:
        radius (float): 圆的半径
    返回值:
        float: 圆的面积
    """
    area = math.pi * radius ** 2
    return area  # 返回计算结果

# 2. 函数调用：计算半径为5的圆的面积
my_area = calculate_circle_area(5)
print(f"半径为5的圆的面积是: {my_area:.2f}")

# 再次调用，计算半径为10的圆的面积
another_area = calculate_circle_area(10)
print(f"半径为10的圆的面积是: {another_area:.2f}")
```

**输出结果：**
```
半径为5的圆的面积是: 78.54
半径为10的圆的面积是: 314.16
```

#### 4. 常见误区

-   **误区一：忘记加括号 `()`**
    很多初学者会写成 `make_coffee` 而不是 `make_coffee()`。不加括号，Python 不会执行函数，而是返回函数对象本身（比如 `<function make_coffee at 0x...>`）。

    ```python
    # 错误示例
    make_coffee  # 这不会执行函数，只是引用函数对象
    ```

-   **误区二：混淆参数和实参**
    认为函数定义时的参数名（`sugar`）和调用时传入的变量名必须一致。实际上，它们只是值的传递，名字可以不同。

    ```python
    def greet(name):
        print(f"你好, {name}")

    my_name = "小明"
    greet(my_name)  # 正确，实参 my_name 的值 "小明" 被传递给了参数 name
    greet("小红")   # 正确，直接传递字符串字面量
    ```

#### 5. 思考题

假设你正在开发一个游戏，需要为不同角色（如战士、法师、弓箭手）计算攻击力。每个角色的攻击力计算公式不同（例如：战士 = 力量 * 2，法师 = 智力 * 3，弓箭手 = 敏捷 * 2.5）。如果不使用函数，你需要在代码中重复写三次不同的计算公式。请思考，如何利用函数来优雅地解决这个问题，使得代码更简洁、更易于维护？
