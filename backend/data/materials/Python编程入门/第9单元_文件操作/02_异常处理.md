## 第9单元 文件操作

### 知识点：异常处理

#### 1. 核心概念

想象一下，你正在厨房里按照食谱做一道菜。食谱上写着：“将鸡蛋打入碗中”。但当你拿起鸡蛋时，发现它已经坏了，一打开就散发出难闻的气味。这时，你会怎么做？是继续把坏鸡蛋倒进碗里，还是停下来处理这个意外情况？

在编程中，程序就像食谱，而“坏鸡蛋”就是程序运行时可能遇到的**异常**。例如，你让程序打开一个不存在的文件，或者让程序计算一个数字除以0。如果程序不处理这些异常，它就会直接“崩溃”并停止运行，就像你无视坏鸡蛋继续做菜，最终毁掉整道菜一样。

**异常处理**就是程序为了应对这些意外情况而预先准备好的“应急预案”。它允许程序在遇到错误时，不直接崩溃，而是执行一段特定的代码来处理错误，然后优雅地继续运行或安全地退出。这大大提高了程序的**健壮性**和**用户友好性**。

#### 2. 关键定义

-   **异常（Exception）**：程序运行过程中发生的、会中断正常执行流程的意外事件。例如，`FileNotFoundError`（文件未找到）、`ZeroDivisionError`（除零错误）。
-   **异常处理（Exception Handling）**：一种编程机制，用于预测、捕获和处理程序运行时可能发生的异常。
-   **`try...except` 语句**：Python中实现异常处理的核心结构。
    -   `try` 块：放置可能引发异常的代码。
    -   `except` 块：当 `try` 块中的代码引发指定异常时，执行此块中的代码。
-   **`else` 子句**：可选的子句，当 `try` 块中的代码没有引发任何异常时执行。
-   **`finally` 子句**：可选的子句，无论是否发生异常，其中的代码都会**始终执行**。常用于执行清理工作，如关闭文件。
-   **抛出异常（Raise an Exception）**：使用 `raise` 关键字主动触发一个异常。

#### 3. 示例与应用

**生活例子 1：自动取款机（ATM）取款**

-   **正常流程**：你插入银行卡 -> 输入密码 -> 输入金额 -> 取出现金。
-   **异常场景**：你输入了错误的密码。
-   **异常处理**：ATM程序不会直接崩溃，而是会捕获这个“密码错误”的异常，然后显示“密码错误，请重试”的提示信息，并允许你再次输入。如果连续多次错误，程序可能会捕获另一个异常，并执行“吞卡”的后续处理。

**生活例子 2：在线购物网站**

-   **正常流程**：你搜索商品 -> 加入购物车 -> 提交订单 -> 支付成功。
-   **异常场景**：当你提交订单时，网络突然中断。
-   **异常处理**：网站程序会捕获“网络连接错误”的异常。它不会让你看到一片空白或错误代码，而是会显示一个友好的提示：“网络连接不稳定，请稍后重试”，并可能自动保存你的购物车信息，等待网络恢复后继续操作。

**Python 代码示例：安全地打开文件**

```python
# 假设我们要打开一个名为 'data.txt' 的文件
filename = 'data.txt'

try:
    # 尝试打开文件
    with open(filename, 'r', encoding='utf-8') as file:
        content = file.read()
        print("文件内容读取成功！")
        print(content)
except FileNotFoundError:
    # 如果文件不存在，执行这里的代码
    print(f"错误：文件 '{filename}' 未找到。请检查文件名。")
except PermissionError:
    # 如果没有权限访问文件
    print(f"错误：没有权限读取文件 '{filename}'。")
except Exception as e:
    # 捕获其他所有类型的异常，并打印错误信息
    print(f"发生了未知错误：{e}")
else:
    # 如果没有发生任何异常，执行这里的代码
    print("文件操作顺利完成，没有发生任何错误。")
finally:
    # 无论是否发生异常，都会执行这里的代码
    print("程序执行完毕，正在清理资源。")
```

#### 4. 常见误区

1.  **滥用 `except` 捕获所有异常**：很多初学者会写一个空的 `except:` 来捕获所有错误，就像这样：
    ```python
    try:
        # 一些代码
    except:
        pass  # 什么都不做
    ```
    **错误理解**：认为这样可以“万事大吉”，程序永远不会崩溃。
    **正确做法**：这种做法非常危险！它会隐藏所有错误，包括你意想不到的严重错误（如内存不足、程序逻辑错误等）。正确的做法是**只捕获你预期会发生的、并且知道如何处理的特定异常**（如 `FileNotFoundError`），或者至少打印出错误信息，以便调试。

2.  **混淆 `else` 和 `finally` 的作用**：
    -   `else` 块只在 `try` 块**没有**异常时执行。
    -   `finally` 块**无论**有没有异常都会执行。
    **错误理解**：认为 `else` 和 `finally` 的作用相同，或者认为 `finally` 只在有异常时执行。
    **正确理解**：`finally` 是“无论如何都要做”的，常用于释放资源（如关闭文件、数据库连接）。`else` 是“一切顺利时才做”的，用于执行依赖于 `try` 块成功完成的后续代码。
