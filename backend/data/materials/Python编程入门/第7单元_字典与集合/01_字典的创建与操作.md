## 第7单元 字典与集合

### 7.1 字典的创建与操作

#### 1. 核心概念

想象一下，你有一本非常特别的通讯录。这本通讯录不是按页码顺序排列的，而是通过一个“标签”来快速找到对应的信息。比如，你只要说出“张三”，就能立刻翻到他的电话号码；说出“李四”，就能找到他的地址。这个“标签”就是**键（Key）**，而找到的信息就是**值（Value）**。

在Python中，**字典（Dictionary）** 就是这样一个强大的数据结构。它存储的是一组组**键值对（Key-Value Pair）**。每个键都是唯一的，就像通讯录里的名字不能重复一样。通过键，我们可以非常快速、高效地访问、修改或删除与之关联的值。字典是无序的（在Python 3.7+版本中，字典会保持插入顺序，但不应依赖此特性进行排序操作），它不关心元素的先后位置，只关心“键”与“值”的对应关系。

#### 2. 关键定义

-   **字典（Dictionary）**：一种可变、无序的容器模型，用于存储键值对。用花括号 `{}` 表示。
-   **键（Key）**：字典中的唯一标识符，用于查找对应的值。键必须是**不可变类型（Immutable Type）**，例如字符串（`str`）、数字（`int`、`float`）或元组（`tuple`）。
-   **值（Value）**：与键相关联的数据。值可以是任意类型的Python对象，包括数字、字符串、列表，甚至是另一个字典。
-   **键值对（Key-Value Pair）**：字典中的基本组成单元，用冒号 `:` 连接键和值，多个键值对之间用逗号 `,` 分隔。例如：`"name": "小明"`。
-   **可变类型（Mutable Type）**：可以在原地修改内容的数据类型，如列表、字典。**注意：键不能是可变类型**，因为如果键的内容变了，字典就无法再正确地找到它了。

#### 3. 示例与应用

**生活例子1：学生成绩查询系统**

假设我们要存储一个班级里学生的数学成绩。用字典来表示就非常直观：

```python
# 创建一个字典，键是学生姓名（字符串），值是成绩（整数）
scores = {
    "小明": 95,
    "小红": 88,
    "小刚": 92
}

# 查询小红的成绩
print(scores["小红"])  # 输出：88

# 修改小明的成绩
scores["小明"] = 98
print(scores["小明"])  # 输出：98

# 添加一个新同学的成绩
scores["小丽"] = 100
print(scores)  # 输出：{'小明': 98, '小红': 88, '小刚': 92, '小丽': 100}
```

**生活例子2：商品库存管理**

一个简单的电商库存系统，用字典来记录每种商品的库存数量。

```python
# 键是商品编号（字符串），值是库存数量（整数）
inventory = {
    "A001": 50,
    "B002": 120,
    "C003": 0
}

# 检查商品A001的库存
print(f"商品A001的库存为：{inventory['A001']}")  # 输出：商品A001的库存为：50

# 商品C003到货，更新库存
inventory["C003"] = 30
print(f"商品C003的库存更新为：{inventory['C003']}")  # 输出：商品C003的库存更新为：30
```

#### 4. 常见误区

-   **误区一：试图用值来查找键。** 字典的设计初衷是“通过键找值”，而不是反过来。虽然可以通过循环遍历实现“通过值找键”，但效率非常低，且不是字典的典型用法。请记住，**键是查找的“钥匙”**。
-   **误区二：使用可变类型作为键。** 初学者可能会尝试用列表作为键，例如 `my_dict = {[1, 2]: "value"}`。这会导致 `TypeError: unhashable type: 'list'` 错误。因为列表是可变类型，其内容可以被修改，一旦修改，字典就无法再通过这个键找到对应的值了。**键必须是不可变的**，如字符串、数字或元组。
