## 第5单元 数据结构

### 5.2 元组与字典

在Python中，除了列表，还有两种非常实用的数据结构：**元组**和**字典**。它们帮助我们以不同的方式组织和存储数据。

#### 1. 核心概念

想象一下，你有一个**只读的清单**，比如一年中固定不变的法定节假日。这个清单一旦写好，就不能再修改、添加或删除任何项目。在Python中，**元组**就是这样一个“不可变”的列表。

再想象一下，你有一本**电话簿**。电话簿里，每个联系人的名字（**键**）都对应着一个电话号码（**值**）。你不能通过“第几个”来查找，而是通过“名字”来查找。在Python中，**字典**就是这样一个“键-值对”的集合，它允许我们通过一个唯一的“键”来快速找到对应的“值”。

#### 2. 关键定义

-   **元组 (Tuple)**：一种有序的、**不可变**的序列。它使用圆括号 `()` 来定义，元素之间用逗号分隔。一旦创建，元组中的元素就不能被修改、添加或删除。
    -   **关键词**：**不可变**、**有序**、**圆括号**。
-   **字典 (Dictionary)**：一种无序的、**可变**的映射类型。它使用花括号 `{}` 来定义，每个元素都是一个**键-值对**，格式为 `键: 值`。通过**键**来访问对应的**值**，键必须是**不可变**类型（如字符串、数字、元组），且**唯一**。
    -   **关键词**：**键-值对**、**可变**、**无序**、**花括号**、**键唯一**。

#### 3. 示例与应用

1.  **生活例子：固定不变的坐标**
    假设你想存储一个地点的经纬度坐标，比如天安门的坐标 `(39.9042, 116.4074)`。这个坐标是固定的，不应该被修改。使用元组非常合适。
    ```python
    # 定义一个元组存储坐标
    tiananmen_coords = (39.9042, 116.4074)
    print(tiananmen_coords)  # 输出: (39.9042, 116.4074)
    # 尝试修改会报错
    # tiananmen_coords[0] = 40.0  # 这行代码会引发 TypeError
    ```

2.  **生活例子：学生信息表**
    假设你要存储一个学生的信息，比如姓名、年龄和学号。使用字典可以清晰地用“键”来描述每个数据的含义。
    ```python
    # 定义一个字典存储学生信息
    student = {
        "name": "张三",
        "age": 18,
        "student_id": "2024001"
    }
    # 通过键来访问信息
    print(student["name"])  # 输出: 张三
    print(student["age"])   # 输出: 18
    ```

3.  **应用场景：函数返回多个值**
    当函数需要返回多个相关的、不可变的值时，元组是默认的选择。例如，计算一个数的商和余数。
    ```python
    def divide(a, b):
        quotient = a // b
        remainder = a % b
        return quotient, remainder  # 实际上返回了一个元组 (quotient, remainder)

    result = divide(10, 3)
    print(result)  # 输出: (3, 1)
    q, r = divide(10, 3)  # 元组解包，分别赋值给q和r
    print(f"商: {q}, 余数: {r}") # 输出: 商: 3, 余数: 1
    ```

#### 4. 常见误区

1.  **混淆元组与列表的语法**
    -   **错误理解**：认为只要用了圆括号就是元组，用了方括号就是列表。
    -   **正确理解**：虽然元组通常用 `()` 定义，但关键区别在于**不可变性**。一个没有逗号的单个元素 `(5)` 实际上是一个整数，而不是元组。要创建单元素元组，必须加逗号，如 `(5,)`。

2.  **试图修改元组**
    -   **错误理解**：像操作列表一样，尝试修改元组中的元素。
    -   **正确理解**：元组是不可变的。任何试图修改其内容的操作（如 `tuple[0] = new_value`）都会导致 `TypeError`。如果需要修改，请考虑使用列表。
