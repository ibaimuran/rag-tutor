## 第4单元 流程控制

### 4.3 while循环与循环控制

#### 1. 核心概念

想象一下，你正在玩一个“猜数字”游戏：计算机随机想一个1到100之间的数字，你需要不断猜测，直到猜中为止。这个“不断猜测”的过程，在编程中就需要用到**循环**。

**`while`循环**是Python中最基本的循环结构之一。它的核心思想是：**“当某个条件成立时，就一直重复执行某段代码”**。就像猜数字游戏，只要“你猜的数字不等于正确答案”这个条件成立，游戏就会让你继续猜。

与`for`循环（通常用于遍历一个已知的序列，如列表）不同，`while`循环更擅长处理**不确定循环次数**的情况。你不知道用户需要猜多少次才能猜中，所以用`while`循环非常合适。

#### 2. 关键定义

-   **`while`循环**：一种基于**条件**的循环结构。只要指定的**条件表达式**为真（`True`），就会重复执行循环体内的代码块。
-   **条件表达式**：一个计算结果为布尔值（`True`或`False`）的表达式。它是`while`循环的“开关”，决定了循环是否继续。
-   **循环体**：被`while`语句控制、需要重复执行的代码块。通常通过**缩进**（4个空格或一个Tab键）来标识。
-   **循环控制**：指在循环执行过程中，通过特定语句来改变循环的正常流程。主要包括：
    -   **`break`语句**：**立即终止**当前整个循环，跳出循环体，执行循环后面的代码。就像猜中数字后，游戏立刻结束。
    -   **`continue`语句**：**跳过本次循环**中剩余的代码，直接进入下一次循环的条件判断。就像在猜数字时，如果你输入了一个不合法的数字（比如负数），程序会忽略这次猜测，直接让你重新输入。
-   **无限循环**：当`while`后面的条件永远为`True`时，循环会无休止地执行下去。除非程序被强制终止或内部有`break`语句，否则会一直运行。这通常是编程错误，但有时也被有意用于需要持续运行的程序（如服务器）。

#### 3. 示例与应用

**例子1：猜数字游戏（`break`的应用）**

```python
import random

target = random.randint(1, 10)  # 随机生成一个1-10的数字
guess = 0

print("猜一个1到10之间的数字：")

while True:  # 创建一个无限循环，等待用户猜中
    guess = int(input("请输入你的猜测："))
    if guess == target:
        print("恭喜你，猜对了！")
        break  # 猜中后，立即跳出循环
    elif guess < target:
        print("猜小了，再试试。")
    else:
        print("猜大了，再试试。")
```

**例子2：用户输入验证（`continue`的应用）**

```python
print("请输入你的年龄（输入-1退出）：")

while True:
    age_input = input("年龄：")
    if age_input == "-1":
        print("程序结束。")
        break
    # 尝试将输入转换为整数
    if not age_input.isdigit():  # 检查输入是否全是数字
        print("输入无效，请输入一个数字。")
        continue  # 跳过本次循环剩余部分，重新开始下一次循环
    age = int(age_input)
    if age < 0 or age > 150:
        print("输入的年龄不合理，请重新输入。")
        continue  # 同样跳过，重新输入
    print(f"你的年龄是：{age}岁。")
    # 处理完有效输入后，可以再次循环或退出
```

**例子3：倒计时（简单的条件循环）**

```python
count = 5
while count > 0:
    print(f"倒计时：{count}秒")
    count -= 1  # 每次循环将count减1，这是关键，否则会无限循环
print("时间到！")
```

#### 4. 常见误区

1.  **忘记更新循环条件，导致无限循环**：
    -   **错误理解**：认为`while`循环会自动结束。
    -   **正确理解**：必须在循环体内改变条件表达式中涉及的变量，否则条件永远为真，循环将无限执行。例如，在例子3中，如果忘记写`count -= 1`，程序将永远打印“倒计时：5秒”。

2.  **混淆`break`和`continue`的作用范围**：
    -   **错误理解**：认为`continue`会结束整个循环。
    -   **正确理解**：`break`是“跳出”整个循环，循环彻底结束；`continue`是“跳过”当前这一次循环的剩余部分，然后**回到循环开头**重新判断条件。`continue`并不会终止循环。
