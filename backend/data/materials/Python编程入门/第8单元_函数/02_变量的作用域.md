## 第8单元 函数

### 8.3 变量的作用域

#### 1. 核心概念

想象一下，你在一栋大楼里工作。大楼的**公共区域**（比如大厅、走廊）所有人都能自由进出和使用。而每个**独立的办公室**，只有拥有该办公室钥匙的员工才能进入，并在里面存放自己的私人物品。

在Python程序中，变量也遵循类似的规则。一个变量并不是在程序的任何地方都能被访问的。**变量的作用域**（Scope）就是指一个变量在程序中可以被访问的“有效范围”或“可见区域”。

简单来说，它定义了“这段代码里，我能看到并使用哪些变量”。理解作用域，能帮助我们避免变量名冲突，写出更清晰、更健壮的代码。

#### 2. 关键定义

- **局部作用域（Local Scope）**：在**函数内部**定义的变量，拥有局部作用域。它只能在该函数内部被访问。函数就像一个“独立办公室”，里面的变量是“私人物品”。
- **全局作用域（Global Scope）**：在**所有函数之外**（即代码的顶层）定义的变量，拥有全局作用域。它可以在程序的任何地方被访问，就像大楼的“公共区域”。
- **全局变量（Global Variable）**：在全局作用域中定义的变量。
- **局部变量（Local Variable）**：在局部作用域（函数内部）中定义的变量。
- **`global` 关键字**：当我们需要在函数内部**修改**一个全局变量的值时，必须使用 `global` 关键字进行声明。否则，Python会认为你在函数内部创建了一个同名的局部变量。

#### 3. 示例与应用

**生活例子1：教室里的“粉笔”**

- **全局变量**：教室里的“公用粉笔”。任何老师（任何函数）都可以使用它。
- **局部变量**：张老师（一个函数）自己带来的“私用红笔”。只有张老师自己能用，李老师（另一个函数）不能碰。

**生活例子2：游戏中的“金币”**

- **全局变量**：游戏中的“总金币数”。所有角色（所有函数）都能看到并尝试使用它。
- **局部变量**：角色A（一个函数）在战斗中临时获得的“增益效果”（如攻击力+10）。这个效果只在该次战斗（该函数）中有效，战斗结束就消失了。

**代码示例：**

```python
# 全局变量
total_score = 0

def add_score(points):
    # 局部变量
    bonus = 10
    # 尝试修改全局变量，需要使用 global 关键字
    global total_score
    total_score = total_score + points + bonus
    print(f"函数内部：本次获得 {points + bonus} 分，总分是 {total_score}")

def show_bonus():
    # 这里无法访问 add_score 函数内部的局部变量 bonus
    # print(bonus)  # 这行代码会报错！
    print("在 show_bonus 函数中，无法看到 bonus 变量。")

# 调用函数
add_score(50)  # 输出：函数内部：本次获得 60 分，总分是 60
show_bonus()   # 输出：在 show_bonus 函数中，无法看到 bonus 变量。

# 在全局作用域中，可以访问全局变量 total_score
print(f"函数外部：最终总分是 {total_score}") # 输出：函数外部：最终总分是 60
```

#### 4. 常见误区

- **误区一：认为函数内部可以随意修改全局变量。**
    - **错误理解**：在函数内部直接写 `total_score = total_score + 10`，以为能修改外部的 `total_score`。
    - **正确理解**：在函数内部，如果没有用 `global` 声明，`total_score = ...` 实际上是创建了一个新的**局部变量**，它和全局的 `total_score` 只是名字相同，但完全是两个不同的变量。这会导致程序逻辑错误，且难以调试。

- **误区二：混淆“读取”和“修改”全局变量。**
    - **错误理解**：认为在函数内部“读取”全局变量的值也需要 `global` 关键字。
    - **正确理解**：在函数内部**读取**一个全局变量的值（比如 `print(total_score)`）是允许的，不需要 `global`。`global` 关键字只在需要**修改**（赋值）全局变量时才必须使用。
