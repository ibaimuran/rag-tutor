## 第7单元 文件与异常处理

### 7.1 文件读写操作

#### 1. 核心概念

想象一下，你正在写一篇日记。你可以在日记本上写下文字（**写入**），也可以翻开之前的页面阅读（**读取**）。在Python程序中，**文件读写操作**就是让程序与计算机硬盘上的文件进行“对话”的过程。

程序运行时，数据通常存储在内存中，一旦程序关闭，这些数据就会消失。而文件是存储在硬盘上的，可以长久保存。因此，文件操作的核心目的就是**实现数据的持久化存储**，让程序能够把运行结果保存下来，或者在下一次运行时读取之前保存的数据。

Python提供了一套简单而强大的函数和方法，让我们可以像操作文本一样，轻松地打开、读取、写入和关闭文件。

#### 2. 关键定义

-   **文件对象 (File Object)**：通过`open()`函数创建的一个对象，它代表了与一个打开文件的连接。所有后续的读写操作都是通过这个对象来完成的。
-   **`open()` 函数**：用于打开一个文件，并返回一个文件对象。它需要至少一个参数：**文件名**。常用的模式有：
    -   `'r'`：**只读模式 (Read)**。默认模式，用于读取文件内容。如果文件不存在，会报错。
    -   `'w'`：**写入模式 (Write)**。用于写入内容。**注意**：如果文件已存在，会清空原有内容并重新写入；如果文件不存在，会创建一个新文件。
    -   `'a'`：**追加模式 (Append)**。用于在文件末尾添加新内容。如果文件不存在，会创建一个新文件。
-   **`read()` 方法**：从文件对象中读取所有内容，返回一个字符串。
-   **`readline()` 方法**：从文件对象中读取一行内容，返回一个字符串（包含行尾的换行符）。
-   **`write()` 方法**：将指定的字符串内容写入文件对象。
-   **`close()` 方法**：关闭文件对象，释放系统资源。**非常重要**，忘记关闭文件可能导致数据丢失或资源泄漏。
-   **`with` 语句**：一种更安全、更简洁的文件操作方式。它会自动管理文件的打开和关闭，即使发生异常也能确保文件被正确关闭。

#### 3. 示例与应用

1.  **记录游戏最高分**
    你开发了一个小游戏，需要把玩家的最高分保存下来，以便下次游戏时读取并显示。
    ```python
    # 写入最高分
    with open('high_score.txt', 'w') as file:
        file.write('1000')

    # 读取最高分
    with open('high_score.txt', 'r') as file:
        score = file.read()
        print(f'最高分是：{score}')
    ```

2.  **保存程序配置**
    你的程序允许用户自定义主题颜色。你可以将用户的偏好（如`dark_theme: True`）保存到一个配置文件中。
    ```python
    # 保存配置
    config = 'dark_theme: True\nfont_size: 14'
    with open('config.ini', 'w') as file:
        file.write(config)

    # 读取配置
    with open('config.ini', 'r') as file:
        for line in file:
            print(line.strip()) # 逐行读取并打印
    ```

3.  **记录日志**
    你的程序在运行过程中，需要记录关键事件或错误信息，方便日后排查问题。
    ```python
    import datetime

    # 追加一条日志
    with open('app.log', 'a') as file:
        file.write(f'{datetime.datetime.now()}: 用户登录成功\n')
    ```

#### 4. 常见误区

-   **误区一：忘记关闭文件**
    很多初学者使用`open()`后，只记得读写，却忘了调用`close()`。这会导致文件资源一直被占用，其他程序可能无法访问该文件，甚至造成数据写入不完整。**强烈建议使用`with`语句**，它能自动处理关闭，避免这个问题。

-   **误区二：混淆写入模式 (`'w'`) 和追加模式 (`'a'`)**
    学生常常以为`'w'`模式是“在文件末尾添加内容”，但实际上它会**覆盖**原有内容。如果需要保留原有内容并添加新内容，必须使用`'a'`模式。这是一个非常容易导致数据丢失的错误。
