## 第4单元 控制流程

### 知识点：循环语句

#### 1. 核心概念

想象一下，你需要抄写100遍“我要好好学习”。如果让你手动复制粘贴100次，是不是很枯燥？在编程中，当我们遇到需要**重复执行**某段代码多次的情况时，循环语句就是我们的“自动抄写员”。

**循环语句**的核心思想是：**让计算机自动、反复地执行一段代码块，直到满足某个特定的条件才停止**。它极大地提高了代码的效率和简洁性，避免了重复编写大量相同或相似的代码。

简单来说，循环就是告诉计算机：“嘿，帮我做这件事，一直做到我说停为止。”

#### 2. 关键定义

- **循环体 (Loop Body)**：被重复执行的代码块。通常用**缩进**来表示。
- **迭代 (Iteration)**：循环体每执行一次，就称为一次迭代。
- **循环条件 (Loop Condition)**：决定循环是否继续执行的**布尔表达式**（结果为`True`或`False`）。当条件为`True`时，循环继续；为`False`时，循环终止。
- **`for` 循环**：一种用于**遍历**（逐个访问）一个序列（如列表、字符串、范围）中每个元素的循环。当你明确知道要循环多少次时，`for`循环非常方便。
- **`while` 循环**：一种只要循环条件为`True`就会持续执行的循环。当你**不确定**循环次数，但知道何时应该停止时，`while`循环更合适。
- **无限循环 (Infinite Loop)**：循环条件永远为`True`，导致循环永远不会停止。这是编程中需要避免的严重错误。

#### 3. 示例与应用

**生活例子1：数羊睡觉**

- **场景**：你睡不着，决定从1数到100只羊。
- **编程对应**：使用`for`循环，明确知道要循环100次。
- **代码示例**：
  ```python
  # 从1数到100只羊
  for sheep_number in range(1, 101):  # range(1,101) 生成1到100的数字序列
      print(f"第{sheep_number}只羊...")
  print("终于睡着了！")
  ```

**生活例子2：检查作业**

- **场景**：老师要检查全班同学的作业，直到所有同学的作业都合格为止。老师不知道需要检查几轮。
- **编程对应**：使用`while`循环，循环条件是“还有同学的作业不合格”。
- **代码示例**：
  ```python
  homework_qualified = False  # 假设一开始作业都不合格
  while not homework_qualified:
      # 检查作业（这里用简单的判断模拟）
      print("老师正在检查作业...")
      # 假设检查后，作业合格了
      homework_qualified = True
  print("所有同学作业都合格了！")
  ```

**应用场景：计算1到100的总和**

- **场景**：计算1+2+3+...+100的结果。
- **代码示例**：
  ```python
  total = 0
  for number in range(1, 101):
      total = total + number  # 每次循环，将当前数字累加到total中
  print(f"1到100的总和是：{total}")
  ```

#### 4. 常见误区

- **误区一：忘记更新循环变量，导致无限循环**
  - **错误理解**：在`while`循环中，循环条件里用到的变量（循环变量）在循环体内没有改变，导致条件永远为`True`。
  - **示例**：
    ```python
    count = 1
    while count <= 5:
        print(count)
        # 忘记写 count = count + 1 这行代码，count永远为1，循环永远执行
    ```
  - **正确做法**：确保在循环体内有改变循环条件的语句，例如 `count = count + 1`。

- **误区二：混淆`for`循环和`while`循环的使用场景**
  - **错误理解**：在需要明确知道循环次数时，错误地使用`while`循环，导致代码复杂且容易出错。
  - **示例**：打印1到10，用`while`循环需要手动初始化变量和更新变量，而用`for`循环则更简洁直观。
  - **正确做法**：**遍历**已知序列或明确次数用`for`；**条件驱动**、次数未知用`while`。
